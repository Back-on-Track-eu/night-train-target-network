# Night Train — Composition Cost Model Calibration

**Goal:** replace the placeholder composition/operator cost values in
`seed.py` with sourced, defensible values — and propose a set of **standard
compositions** (coach lists, places per class, new-build vs. refurbished
material) whose cost parameters fill the tables `service_classes`,
`operators`, `operator_class_costs`, `coach_types`, `coach_type_classes`,
`composition_types`, and `composition_type_coaches`. No schema change; the
deliverable is paste-ready seed value blocks.

**Related documentation:** the evaluation model these rates feed —
[`../../evaluation/README.md`](../../evaluation/README.md) · database seed
receiving the emitted blocks — [`../../../db/README.md`](../../../db/README.md)

**Scope:** operator-controllable costs only. Track access charges, station
access, shunting, parking, and traction energy (consumption and price) are
**excluded** — separate calibration workstreams. Energy regression factors
are passed through unchanged.

---

## Structure: sources → observations → calibration

Earlier drafts split the work into separate bottom-up and top-down notebooks.
That split was dropped: our sources don't divide that way — the strongest
ones (nox model, ÖBB annual report, Schönerstedt/Standardisierte Bewertung)
each deliver *both* per-parameter rates and aggregate benchmarks. Organising
by method would mean opening every source twice. Instead:

```
notebooks/
├── 01_source_extraction.ipynb   one section per source; every extracted
│                                  number becomes a row in data/*.csv with
│                                  full provenance
├── 02_calibration.ipynb          the model: standard compositions, rate
│                                  calibration, cost structure computation,
│                                  benchmark validation, seed emission
└── data/                         generated by 01, checked in (diffable
                                   provenance record)
    ├── sources_register.csv          one row per source: publisher, pub
    │                                   year, PRICE BASIS year, currency,
    │                                   reliability note
    ├── parameter_observations.csv    one row per (parameter, source)
    ├── benchmark_observations.csv    aggregates per (category, source):
    │                                   €/train-km, ct/place-km, €/trip,
    │                                   shares
    ├── cost_bridge_mapping.csv       source account → our formula key,
    │                                   with in_scope flag and adjustment
    │                                   note
    └── coach_procurement_datapoints.csv  (price, coach category, condition,
                                        places, year) tuples for the
                                        purchase-price fit
```

The literals in notebook 01 are the master; the CSVs are generated artifacts.
Bottom-up and top-down logic both live in notebook 02 as consecutive sections
over the same observation tables, so reconciliation is a dataframe join.

## The calibration model (notebook 02)

**Input:** standard composition proposals — ordered coach list, places per
service class, gross weight, max speed, and **material strategy** (new-build
vs. refurbished). Plus a parametrised reference route (needed only to turn
rates into comparable aggregates).

**Output:** the differentiated cost structure per composition — per year, per
trip, per train-km, per place-km, and category shares — and paste-ready seed
blocks for all target tables.

**New-build vs. refurbished** enters through four condition-dependent rates:
purchase cost per coach, amortisation period (full life vs. remaining life),
availability, and per-coach-km maintenance. The four proposed standard
compositions cover both strategies (refurbished mixed / refurbished
capacity / new-build mini-cabin / new-build premium), patterned on real
fleets (European Sleeper, Nightjet new generation) with capacities marked
for verification during extraction.

**Validation** runs on three normalisations simultaneously — €/train-km per
category (absolute rate errors), % share of total (structural errors), and
ct/place-km (composition-scaling errors) — against the bridged benchmark
corridor, ±30% acceptance band. The UIC composition spread (seat-heavy
cheapest, premium sleeper-heavy dearest per place-km) is asserted as an
ordering check.

## Unit audit findings (must-know before touching values)

- **`coach_maint_eur_km`:** the schema comment says €/coach-km, but
  `models/evaluation/calc.py` applies the stored value per **train-km** with
  no coach-count multiplier. Resolution (no code change): the notebook
  calibrates a per-coach-km rate and emits `rate × n_coaches` per
  composition type.
- **`purchase_coach_eur`** is one value per composition; real prices differ
  per coach type. Emission writes the fleet-weighted average across the
  composition's coach list.
- **`driver/crew_overhead_h`** are loaded but not yet applied in the backend
  (known TODO). The notebook's driver layer applies them, since the rates
  are calibrated for the intended formula.
- The notebook mirrors the operator-controllable subset of `CALC_FORMULAS`
  deliberately (self-contained, no backend imports). If backend formulas
  change, update the mirror in section 2 of notebook 02.

## Backend follow-up: `fix_overhead_eur` formula change (decided 2026-07-10)

The notebook already computes fixed overhead as a share of all other
operating costs. The backend still implements share-of-own-amortisation
(`calc.py`, `CompositionFleetCost`). Required change, to be made when the
calibrated seeds land:

- Move the computation out of `CompositionFleetCost` (it can no longer be
  computed at fleet level — it needs the full cost picture) to the
  aggregation level where all cost leaves are known.
- **Base definition:** sum of operator cost leaves only — driver, crew,
  loco, coach maintenance, cleaning, amortisation, financing, stockings —
  **excluding** fix_overhead itself, var_overhead, ebit_margin, and all
  infrastructure/energy leaves (tac, energy, station, parking, shunting).
  The 0.12 quota was derived on exactly this base; including infrastructure
  would over-produce overhead.
- Update `CALC_FORMULAS["fix_overhead_eur"]` (LaTeX + description) and the
  `operators` DDL comment stays as is (it already describes the new
  behaviour).
- Bump `CALC_VERSION` (0.9.2 → 0.10.0) with changelog entry; breakdown tree
  placement of the fix_overhead node changes → heads-up to Bjarne (values
  and node location change; response shape otherwise unchanged).
